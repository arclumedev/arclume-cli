# Arclume CLI

Configure any repository for optimal Arclume indexing. Run it once to give Arclume the context it can't infer from code structure alone — architecture, intent, entry points, and what to ignore.

**[arclume.dev](https://arclume.dev)** · **[Documentation](https://arclume.dev/docs)**

---

## What is Arclume?

Arclume is a codebase intelligence platform that builds a structural model of your entire software system — parsing functions, mapping dependencies, and resolving cross-service connections using AST-level analysis and dependency graph traversal.

It powers two things:

- **Planning:** Convert briefs or meeting transcripts into implementation-ready stories with actual file paths, line numbers, and dependency mappings — pushed directly to Linear.
- **Building:** An MCP server for Cursor, Claude Code, and VS Code that answers live cross-repository architecture questions without leaving your editor.

The CLI is how you configure a repo for indexing — giving Arclume the architectural context that makes both of these accurate.

```bash
# npm
npm install -g arclume

# Homebrew
brew install arclume

pip install arclume
```

## Authentication

```bash
arclume login
```

Opens a browser-based OAuth flow. Once authenticated, your token is stored in the OS keychain (macOS Keychain, Windows Credential Manager, libsecret on Linux) — no passwords, no manual token copying.

One login per machine, not per repo. To check who you're logged in as:

```bash
arclume whoami
```

**CI/CD:** Set `ARCLUME_TOKEN` as an environment variable. Generate a CI token from **Settings → API Tokens** in the Arclume web app.

```yaml
# GitHub Actions
- name: Re-index on merge
  run: arclume index
  env:
    ARCLUME_TOKEN: ${{ secrets.ARCLUME_TOKEN }}
```

## Quickstart

```bash
cd your-repo
arclume init
# review and edit .arclume/context.md
arclume index
```

## Commands

### `arclume init`

One-time setup. Auto-detects your stack, infers entry points, and scaffolds all config files with smart defaults. Prompts for anything it can't infer.

```bash
arclume init           # interactive setup
arclume init --dry-run # preview what would be generated
```

**Generated files** (all safe to commit):

```
.arclume/
  config.json     ← indexer configuration
  context.md      ← architectural description (edit this)
.arclumeignore    ← indexing-specific ignore patterns
openspec/
  specs/
    OVERVIEW.md   ← spec stub (if OpenSpec not already present)
```

---

### `arclume context`

Manage `.arclume/context.md` — the primary lever for improving index quality. This file is ingested as a top-level anchor node that shapes how the entire graph relates.

```bash
arclume context validate     # check context.md and config.json are well-formed
arclume context diff         # show changes since last commit
arclume context generate     # AI-draft context.md from the existing index (requires indexed repo)
arclume context generate --spec-delta  # also generate a candidate OpenSpec delta
```

---

### `arclume ignore`

Manage `.arclumeignore` — gitignore-syntax patterns for excluding files from indexing.

```bash
arclume ignore suggest                  # recommend patterns based on detected stack
arclume ignore add "src/generated/**"   # add a pattern directly
arclume ignore list                     # print all active patterns
```

---

### `arclume index`

Trigger indexing via the Arclume API.

```bash
arclume index          # full re-index
arclume index --watch  # watch for file changes and re-index incrementally
arclume index status   # show current index status
```

---

### `arclume doctor`

Diagnose common issues that degrade index quality.

```bash
arclume doctor
```

Checks for: missing `context.md`, no entry points configured, too many files indexed, stale index, OpenSpec present but not wired into config, and more. Prints a colored pass/fail for each check with actionable suggestions.

---

### `arclume login` / `arclume logout` / `arclume whoami`

Manage authentication. See [Authentication](#authentication) above.

## Configuration

### `.arclume/config.json`

Machine-readable configuration consumed by the indexer. Generated by `arclume init`, safe to edit.

```json
{
  "version": "1",
  "language": ["typescript"],
  "framework": ["next.js"],
  "entryPoints": ["src/index.ts", "src/server.ts"],
  "graphDepth": 5,
  "chunkSize": 512,
  "openspec": true,
  "contextFile": ".arclume/context.md",
  "highWeightPaths": ["src/core/", "src/lib/"],
  "lowWeightPaths": ["src/__tests__/", "scripts/"]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `entryPoints` | `string[]` | Where call graph traversal starts — the most important field |
| `graphDepth` | `number` | Hops deep to trace from each entry point |
| `chunkSize` | `number` | Token budget per chunk; tune for repos with very long or short functions |
| `highWeightPaths` | `string[]` | Directories to boost in search ranking |
| `lowWeightPaths` | `string[]` | Directories to suppress in search ranking |
| `openspec` | `boolean` | Elevate `openspec/specs/*.md` as high-weight anchor nodes |

### `.arclume/context.md`

The most impactful thing you can do for index quality. Write a plain-English description of your repo — what it does, how it's structured, key design decisions, and what to de-emphasize. Treat it like an architectural README that Arclume actually reads.

```markdown
## What this is
## Architecture
## Entry Points
## Key Design Decisions
## What to de-emphasize
```

Keep it updated as the codebase evolves. `arclume context generate --spec-delta` can help draft updates after significant changes.

## OpenSpec Integration

If `openspec/` is present in the repo, Arclume treats it as first-class context. Specs become high-weight anchor nodes that shape map generation and module boundaries.

```bash
# After a significant change, generate a candidate spec update
arclume index
arclume context generate --spec-delta
# review openspec/changes/auto-{timestamp}/ and commit if accurate
```

## Suggested Workflow

**Active development:**
```bash
arclume index --watch
```

**After merging a significant change:**
```bash
arclume index
arclume context generate --spec-delta
```

**Periodic health check:**
```bash
arclume doctor
```
